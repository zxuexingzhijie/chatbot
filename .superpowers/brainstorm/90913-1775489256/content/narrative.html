剧情控制与叙事策略

主线+支线+动态生成 三层叙事结构 — 以奇幻酒馆为例

三层叙事结构
🔴 主线剧情 — 设计者预设
不可跳过的关键节点，因果链驱动
1 进入酒馆，遇到格里姆
↓
2 获取地下室线索
↓
3 找到钥匙，打开地下室
↓
4 发现密道秘密
🟡 支线故事 — 条件触发
丰富世界，可选但有奖励
帮助旅行者
触发: 与大厅旅行者对话
奖励: 获得地图碎片
神秘旅客的委托
触发: 进入客房走廊
奖励: 解锁隐藏结局
后院的废弃马车
触发: 搜索后院
奖励: 备用钥匙(主线捷径)
🟢 动态生成 — LLM实时创作
细节填充，每次不同的体验
NPC对话
基于关系值+知识+性格动态生成
环境描写
根据时间/天气/事件状态变化
随机事件
酒馆打架、新旅客到来等
失败叙事
玩家失败时生成替代剧情
剧情节点定义格式 (YAML)
story_nodes:
  - id: "enter_tavern"
    type: main
    description: "玩家首次进入酒馆"
    trigger:
      condition: "player.location == 'tavern_hall' AND NOT events.contains('entered_tavern')"
    effects:
      - add_event: "entered_tavern"
      - set_npc_state: {id: "bartender_grim", mood: "cautious"}
    narrator_hint: "描写昏暗酒馆的氛围，格里姆的第一印象"
    next_nodes: ["get_cellar_clue"]

  - id: "get_cellar_clue"
    type: main
    description: "获取地下室线索"
    trigger:
      any_of:                          # 多路径触发
        - "player.talked_to('bartender_grim') AND bartender_grim.trust >= 20"
        - "player.has_item('torn_notice') AND player.used('torn_notice')"
        - "player.searched('tavern_bar') >= 3"   # 搜索3次也能发现
    effects:
      - add_knowledge: {target: "player", fact: "地下室有密道"}
      - add_event: "cellar_clue_found"
    narrator_hint: "揭示地下室存在秘密的线索，保持悬念"
    fail_forward:                       # 失败推进机制
      after_turns: 20                  # 20回合未触发则自动推进
      fallback_event: "mysterious_noise_from_cellar"
      narrator_hint: "地下室传来奇怪声响，引起玩家注意"

叙事控制机制

硬性约束
• 世界物理规则不可违反
• 主线节点必须按因果链推进
• 锁定条件未满足时拒绝操作
• NPC知识边界（不知道的不说）
• 死亡/重大失败触发存档回滚
软性引导
• NPC对话中暗示下一步方向
• 环境描写突出关键物品/线索
• hint 命令提供非强制建议
• 长时间无进展触发过渡事件
• 失败后给出替代路径（Fail Forward）
连贯性保障
• 事件时间线供LLM参考
• NPC记住与玩家的交互历史
• 伏笔在后续场景中被回收
• 变量填充（角色名/物品名）
• Prompt注入世界状态摘要
