横切模块详细设计

状态管理器 / 记忆系统 / LLM适配器 — 内部实现细节

① 状态管理器 (world/state.py)
类结构
class WorldState(frozen Pydantic BaseModel):
    turn: int
    player: Character
    locations: dict[str, Location]
    characters: dict[str, Character]
    items: dict[str, Item]
    relationships: list[Relationship]
    quests: dict[str, QuestStatus]
    timeline: list[Event]

    def apply(self, diff: StateDiff) -> WorldState:
        """不可变更新 — 返回新WorldState实例
        合并diff中的变更到当前状态
        """

class StateDiff(BaseModel):
    """规则引擎输出的状态变更描述"""
    updated_characters: dict[str, CharacterPatch]
    updated_locations: dict[str, LocationPatch]
    added_items: list[Item]
    removed_items: list[str]
    relationship_changes: list[RelationshipDelta]
    quest_updates: dict[str, QuestStatus]
    new_events: list[Event]
    turn_increment: int = 1

class StateManager:
    _current: WorldState
    _history: deque[WorldState]  # undo栈，maxlen=50
    _redo: deque[WorldState]     # redo栈

    def commit(self, diff: StateDiff) -> WorldState:
        """将diff应用到当前状态
        1. 推入_history栈（保存旧状态）
        2. _current = _current.apply(diff)
        3. 清空_redo栈
        4. 触发auto_save检查
        返回新的WorldState
        """

    def undo(self) -> WorldState:
        """回退一步
        1. 推当前状态到_redo栈
        2. _current = _history.pop()
        """

    def redo(self) -> WorldState:
        """重做一步（如果redo栈非空）"""

    def snapshot(self) -> dict:
        """序列化当前状态为JSON-ready dict"""

    def save(self, name: str, path: Path) -> Path:
        """存档到 saves/{name}.json
        1. snapshot()获取当前状态dict
        2. 附加meta: save_version, timestamp, checksum
        3. 写入JSON文件（atomic write: 先写tmp再rename）
        """

    def load(self, name: str, path: Path) -> WorldState:
        """从 saves/{name}.json 加载
        1. 读取JSON
        2. 校验save_version兼容性
        3. 校验checksum完整性
        4. 反序列化为WorldState
        5. 清空_history和_redo栈
        """

    def list_saves(self, path: Path) -> list[SaveInfo]:
        """列出所有存档，返回名称+时间+回合数"""
设计决策
不可变状态 (Immutable)
WorldState是frozen Pydantic model，每次变更返回新实例。
原因: undo/redo只需保存引用，无需深拷贝；避免引擎和叙事层同时修改同一对象导致竞态；状态快照天然可序列化。
StateDiff 模式
规则引擎不直接修改WorldState，而是输出一个StateDiff描述"要改什么"。StateManager.commit()负责应用。
原因: 变更可审计（日志记录diff）；叙事层可读取diff来描述"发生了什么"；retry时丢弃diff即可。
Undo/Redo 栈
deque(maxlen=50) 限制内存。每次commit推旧状态入history。
undo = history.pop → redo.push
redo = redo.pop → history.push
新commit清空redo栈。
存档安全
• Atomic write: 先写 .tmp 再 os.rename
• Checksum: SHA256校验防止损坏
• Version字段: 支持未来格式迁移
• Auto-save: 每N回合或关键节点自动存档
数据流
规则引擎 → 输出 StateDiff
→ StateManager.commit(diff)
→ 返回新 WorldState
→ 新 WorldState 传给叙事层生成文本
→ 同时 diff 传给叙事层描述变化
② 记忆系统 (world/memory.py)
类结构
class EventTimeline:
    """线性事件时间线 — 记录所有已发生事件"""
    _events: list[Event]  # 按turn排序

    def append(self, event: Event) -> None:
        """追加事件，自动按turn排序"""

    def recent(self, n: int = 5) -> list[Event]:
        """获取最近n个事件，供Prompt注入"""

    def query(self, **filters) -> list[Event]:
        """按条件过滤事件
        filters: actor=, type=, after_turn=, contains=
        例: query(actor="bartender", type="dialogue")
        """

    def summarize(self, max_tokens: int = 200) -> str:
        """生成事件摘要文本（供Prompt使用）
        策略: 最近5条完整 + 更早的压缩为1句概述
        用于控制Prompt长度
        """

    def has(self, event_id: str) -> bool:
        """检查某事件是否已发生
        供剧情引擎的trigger条件使用
        """

class RelationshipGraph:
    """角色关系有向图"""
    _edges: dict[tuple[str,str], Relationship]

    def get(self, src: str, tgt: str) -> Relationship | None:
        """获取两个角色间的关系"""

    def update(self, delta: RelationshipDelta) -> Relationship:
        """更新关系值（不可变 — 返回新Relationship）
        delta: {source, target, type, change: +/-int}
        自动clamp到[-100, 100]
        如果关系不存在则创建（初始值=0+change）
        """

    def get_all_for(self, char_id: str) -> list[Relationship]:
        """获取某角色的所有关系
        供NPC对话时参考态度
        """

    def describe_for_prompt(self, char_id: str) -> str:
        """生成角色关系描述文本
        例: '格里姆对你态度冷淡(trust:-20)'
        供叙事Prompt注入
        """

class KnowledgeBase:
    """角色知识库 — 记录每个角色"知道什么" """
    _knowledge: dict[str, frozenset[str]]

    def knows(self, char_id: str, fact: str) -> bool:
        """检查角色是否知道某个事实
        供规则引擎限制NPC不泄露未知信息
        """

    def teach(self, char_id: str, fact: str) -> None:
        """让角色学会某个事实
        不可变: 创建新frozenset替换旧的
        """

    def facts_for_prompt(self, char_id: str) -> str:
        """生成角色知识列表
        供叙事Prompt约束NPC对话边界
        """

class MemorySystem:
    """统一入口 — 组合以上三个子系统"""
    timeline: EventTimeline
    relationships: RelationshipGraph
    knowledge: KnowledgeBase

    def build_context(self, actor: str, max_tokens: int) -> MemoryContext:
        """为Prompt构建完整记忆上下文
        组合: 最近事件 + 相关关系 + 角色知识
        控制总token数不超过max_tokens
        优先级: 最近事件 > 直接关系 > 知识
        返回MemoryContext(可直接嵌入Prompt)
        """

    def apply_diff(self, diff: StateDiff) -> None:
        """从StateDiff中提取记忆相关变更
        - 追加new_events到timeline
        - 应用relationship_changes到graph
        - 提取knowledge变更到knowledge base
        """
设计决策
三子系统分离
Timeline、RelationshipGraph、KnowledgeBase 各自独立，MemorySystem做组合。
原因: 各自有不同的查询模式和数据结构；独立可测试；Prompt构建时按需取用。
Prompt Token预算
build_context() 接受 max_tokens 参数，按优先级填充：
1. 最近5条事件（最重要）
2. 与当前交互相关的关系
3. 当前NPC的知识列表
超出预算则截断低优先级部分。timeline.summarize()负责压缩早期事件。
与StateManager的关系
MemorySystem是WorldState的读视图。
StateManager.commit() 后调用 MemorySystem.apply_diff() 同步。
undo/load时，从新WorldState重建MemorySystem。
序列化
MemorySystem自身不持久化。所有数据来源于WorldState。
存档 = WorldState序列化（含timeline、relationships）。
加载 = 反序列化WorldState → 重建MemorySystem。
单一数据源，避免不一致。
③ LLM适配器 (llm/adapter.py)
类结构
class LLMConfig(BaseModel):
    provider: str          # "openai" | "anthropic" | "ollama"
    model: str             # "gpt-4o-mini"
    temperature: float     # 0.0 ~ 1.0
    max_tokens: int
    base_url: str | None   # Ollama自定义endpoint
    api_key: str | None    # 从环境变量读取
    timeout: float = 30.0
    max_retries: int = 3

class LLMAdapter(Protocol):
    """所有后端必须实现此协议"""

    async def complete(
        self,
        messages: list[Message],
        response_format: type[T] | None = None,
    ) -> T | str:
        """通用补全
        response_format不为None时，返回Structured Output
        (Pydantic model实例)
        为None时，返回纯文本str
        """

    async def stream(
        self,
        messages: list[Message],
    ) -> AsyncIterator[str]:
        """流式输出，yield每个文本chunk"""

class OpenAIAdapter(LLMAdapter):
    """OpenAI实现"""
    def __init__(self, config: LLMConfig):
        self._client = AsyncOpenAI(
            api_key=config.api_key,
            timeout=config.timeout,
        )

    async def complete(self, messages, response_format=None):
        """
        response_format → openai的response_format参数
        使用Pydantic model → JSON Schema自动转换
        内置重试: tenacity, 指数退避, max_retries次
        """

    async def stream(self, messages):
        """
        使用openai stream=True
        async for chunk in response:
            yield chunk.choices[0].delta.content
        """

class AnthropicAdapter(LLMAdapter):
    """Anthropic实现 (P4)
    system message → system参数
    response_format → tool_use模式模拟Structured Output
    stream → async message streaming
    """

class OllamaAdapter(LLMAdapter):
    """Ollama实现 (P4)
    使用httpx直连本地API
    response_format → JSON mode
    stream → Server-Sent Events
    """

class LLMRegistry:
    """适配器工厂与注册中心"""
    _adapters: dict[str, type[LLMAdapter]] = {
        "openai": OpenAIAdapter,
        "anthropic": AnthropicAdapter,
        "ollama": OllamaAdapter,
    }

    def create(self, config: LLMConfig) -> LLMAdapter:
        """根据config.provider创建适配器实例"""

class LLMService:
    """业务层 — 组合两个适配器实例"""
    _intent_llm: LLMAdapter    # 轻量模型
    _narrative_llm: LLMAdapter # 强模型

    async def classify_intent(
        self, input: str, context: SceneContext
    ) -> ActionRequest:
        """
        1. 构建意图分类messages (few-shot + 场景实体)
        2. _intent_llm.complete(messages,
             response_format=ActionRequest)
        3. 返回Pydantic ActionRequest实例
        """

    async def generate_narrative(
        self, action: ActionResult,
        state: WorldState, memory: MemoryContext
    ) -> str:
        """
        1. 用prompts.py构建叙事messages
        2. _narrative_llm.complete(messages)
        3. 返回叙事文本
        """

    async def stream_narrative(
        self, action: ActionResult,
        state: WorldState, memory: MemoryContext
    ) -> AsyncIterator[str]:
        """
        同上但用_narrative_llm.stream()
        Rich渲染层消费iterator实现打字机效果
        """
设计决策
Protocol而非ABC
使用Python Protocol（结构化子类型），不要求继承。
原因: 第三方可直接实现接口而无需导入基类；更Pythonic；利于测试时创建mock。
complete() + Structured Output
single complete()方法同时处理纯文本和JSON模式。
传入response_format（Pydantic model类型）时，自动启用Structured Output返回类型安全的实例。
意图分类: complete(msgs, ActionRequest) → ActionRequest实例
叙事生成: complete(msgs) → str
双实例架构 (LLMService)
LLMService持有两个LLMAdapter实例:
• _intent_llm: 轻量快速（mini/haiku）
• _narrative_llm: 高质量（4o/sonnet）
原因: 意图分类要求低延迟+确定性，叙事要求创造力+质量，两者需求矛盾，分开配置最优。
错误处理与重试
• 网络错误: tenacity指数退避重试（max 3次）
• Rate limit: 自动等待 retry-after
• JSON解析失败: 重试1次，仍失败则返回CUSTOM动作
• 超时: 配置化timeout，默认30s
• 所有错误包装为 LLMError 自定义异常
配置热加载
config.yaml变更时，LLMRegistry重新创建适配器实例。
利用watchdog监听文件变更，或通过游戏内 reload 命令手动触发。
无需重启游戏即可切换模型。
完整配置示例 (config.yaml)
llm:
  intent:
    provider: openai
    model: gpt-4o-mini
    temperature: 0.1        # 低温度 → 确定性分类
    max_tokens: 200          # 意图JSON很短
    max_retries: 3
    timeout: 10.0            # 快速失败

  narrative:
    provider: anthropic
    model: claude-sonnet-4-6
    temperature: 0.8         # 高温度 → 创意叙事
    max_tokens: 500
    max_retries: 2
    timeout: 30.0
    stream: true             # 打字机效果

game:
  auto_save_interval: 5      # 每5回合自动存档
  undo_history_size: 50
  save_dir: ./saves

debug:
  show_intent_json: false    # 调试时显示意图解析结果
  show_prompt: false         # 调试时显示完整Prompt
  log_level: INFO
