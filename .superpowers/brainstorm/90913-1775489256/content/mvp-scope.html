LLM多后端适配器 & MVP开发范围

统一接口设计 + 分阶段交付计划

LLM 适配器架构
LLMAdapter (Protocol / ABC)
async classify_intent(input, context) → ActionRequest
async generate_narrative(action, state, prompt) → str
async stream_narrative(...) → AsyncIterator[str]
───────────────────────
model: str
max_tokens: int
temperature: float
OpenAI
gpt-4o
gpt-4o-mini
Anthropic
sonnet-4.6
haiku-4.5
Ollama
llama3
qwen2 等
配置文件 (config.yaml)
llm:
  intent:          # 意图分类
    provider: openai
    model: gpt-4o-mini
    temperature: 0.1
    max_tokens: 200

  narrative:       # 叙事生成
    provider: anthropic
    model: claude-sonnet-4-6
    temperature: 0.8
    max_tokens: 500
    stream: true
💡 设计决策
• 意图分类和叙事生成可用不同模型
• 叙事生成支持流式输出（打字机效果）
• 配置热加载，无需重启切换模型
• 统一错误处理 + 自动重试
MVP 分阶段交付
P1
Phase 1 — 核心骨架
✅ 项目结构搭建（Python包、依赖管理）
✅ Rich CLI界面（三区布局、输入/输出循环）
✅ 世界模型（Character, Location, Item 数据类）
✅ LLM适配器接口 + OpenAI实现
✅ 意图解析层（LLM分类 → ActionRequest）
✅ 基础规则引擎（位置移动、物品交互验证）
交付物: 可以在酒馆里移动、观察、拾取物品
P2
Phase 2 — 对话与叙事
✅ NPC对话系统（关系值影响对话风格）
✅ LLM叙事生成 + 流式输出（打字机效果）
✅ Prompt模板管理（场景描写、对话、引导）
✅ 记忆系统（事件时间线、NPC知识）
✅ 系统命令实现（look/inventory/status/hint/help）
交付物: 可以和NPC自由对话，感受AI叙事
P3
Phase 3 — 剧情与存档
✅ 剧情节点引擎（YAML定义、条件触发）
✅ 主线任务：酒馆地下室之谜（完整可通关）
✅ 存档/读档功能（JSON序列化）
✅ continue/undo/retry 游戏控制命令
✅ Fail Forward 机制（超时推进）
交付物: 完整可通关的酒馆故事体验

Phase 4 — 扩展与打磨
✅ 支线任务（帮助旅行者、神秘旅客委托）
